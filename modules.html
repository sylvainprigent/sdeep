Modules#

Most of the modules used by SDeep inherit from PyTorch modules. The two modules specific to SDeep are SWorkflow and Eval

SDeep main modules#

Transform

Callable to transform a tensor into another tensor of same shape

Dataset

Read and apply transform to data for training or validation

Module

Neural network as a PyTorch module

Loss

Loss function to be minimized. Implemented as a PyTorch module

Optim

Optimizer for back-propagation. Implemented as a PyTorch optimizer

SEval

Implements the evaluation metrics and write evaluation output files

SWorkflow

Implements the training strategy

Interfaces#

Interfaces are the core of the framework to define modules that store metadata and allows to efficiently build training and prediction workflows.

The available interfaces are:

SDataset

Interface for a dataset compatible with the framework

SEval

Interface for the evaluation modules

SModel

Interface for a model compatible with the framework

STransform

Interface for a data transform compatible with the framework

SWorkflow

Default workflow to train and predict a neural network

Workflows#

SWorkflowBase

Default workflow to train and predict a neural network

ClassificationWorkflow

Workflow to train and predict a classification neural network

RestorationWorkflow

Workflow to train and predict a restoration neural network

SegmentationWorkflow

Workflow to train and predict a semantic segmentation neural network

SelfSupervisedWorkflow

Workflow to train and predict a restoration neural network

Eval#

EvalClassification

Generate an evaluation of classification results

EvalRestoration

Generate an evaluation of denoising results

Models#

Autoencoder

Implementation of image autoencoder network

DnCNN

Implementation of the DnCNN network

DeepFinder

Implementation of a 2D version of the DeepFinder UNet model

DRUNet

Implementation of the DRUNet network

MNistClassifier

Basic classifier for the MNist dataset to test and demo the classification framework

UNet

Implementation of a UNet network

Losses#

Losses listed in the documentation are the one implemented in SDeep. Any PyTorch loss can be made accessible to the SDeep framework using the export=[MyLoss] in the losses module. By default PyTorch available losses are:

  • MSELoss

  • L1Loss

  • BCELoss

  • BCEWithLogitsLoss

  • CrossEntropyLoss

DeconMSEHessian

MSE LOSS with a (de)convolution filter and Hessian regularization

DeconSpitfire

MSE LOSS with a (de)convolution filter and Spitfire regularisation

DiceLoss

Multiclass Dice loss using one hot encode input

BinaryDiceLoss

Dice loss for binary classification

VGGL1PerceptualLoss

VGG based perceptual loss

N2XDenoise

MSE Loss with mask for Noise2Void denoising

N2XDecon

MSE Loss with mask for Noise2Void deconvolution

TverskyLoss

Tversky loss for instance segmentation

Datasets#

MNISTClassif

Dataset to load MNIST dataset for image classification

MNISTAutoencoder

Dataset to use the MNIST data for autoencoders

RestorationDataset

Dataset for image restoration from full images

RestorationPatchDataset

Dataset for image restoration using patches

RestorationPatchDatasetLoad

Dataset for image restoration using patches preloaded in memory

SegmentationDataset

Dataset for image segmentation where images are stored in a single directory

SegmentationPatchDataset

Dataset for image segmentation where images are partitioned into patches

SelfSupervisedDataset

Gray scaled image patch dataset for Self supervised learning

SelfSupervisedPatchDataset

Gray scaled image patch dataset for Self supervised learning

Transforms#

FlipAugmentation

Data augmentation by flipping images

RestorationAugmentation

Data augmentation flipping images

VisionScale

Scale images in [-1, 1]

VisionCrop

Crop the images at the center

Optims#

SDeep uses optimizers from PyTorch. Current available optimizers are:

  • Adam

  • SGD
